Descripción del Sistema

Contexto del Problema

Se requiere una plataforma centralizada para gestión de carteras de inversión, registro de operaciones, monitoreo de rendimiento y análisis de mercado en tiempo real.

Funcionalidades Implementadas

Gestión de Portfolios

  • Creación y administración de múltiples carteras
  • Registro de operaciones de compra y venta
  • Cálculo de métricas financieras
  • Persistencia de historial de transacciones

Integración de Servicios

  • Consulta de precios de mercado (Alpha Vantage API)
  • Análisis mediante modelos de lenguaje (OpenAI API)
  • Generación de reportes de rendimiento
  • Autenticación basada en tokens JWT

Características Arquitectónicas

Diseño modular con bajo acoplamiento. Las integraciones externas son opcionales. El núcleo del sistema opera de forma independiente, permitiendo despliegue sin dependencias de APIs de terceros.

API REST - Endpoints Principales

Autenticación y Usuarios

  • POST /api/v1/auth/register - Registro de nuevos usuarios
  • POST /api/v1/auth/login - Autenticación JWT (access + refresh tokens)
  • POST /api/v1/auth/refresh - Renovación de tokens
  • POST /api/v1/auth/logout - Cierre de sesión individual
  • POST /api/v1/auth/logout-all - Cerrar todas las sesiones
  • GET /api/v1/users/me - Perfil del usuario autenticado

Portfolios y Operaciones

Gestión de Carteras

  • GET /api/v1/portfolios
  • POST /api/v1/portfolios
  • GET /api/v1/portfolios/{id}
  • PUT /api/v1/portfolios/{id}

Operaciones & Análisis

  • POST /api/v1/operations
  • GET /api/v1/operations
  • GET /api/v1/market/price/{symbol}
  • POST /api/v1/analysis/generate

Arquitectura del Sistema

Arquitectura en Capas

Implementación modular aplicando principios de separación de responsabilidades (SOLID) y bajo acoplamiento.

Estructura de Capas

1. Capa de Presentación (FastAPI)

Endpoints REST versionados (/api/v1/). Procesamiento de peticiones HTTP, validación de datos de entrada mediante esquemas Pydantic, serialización de respuestas.

2. Capa de Lógica de Negocio

Servicios desacoplados de la infraestructura. Orquestación de operaciones, implementación de reglas de negocio, cálculos de métricas financieras, coordinación con servicios externos.

3. Capa de Acceso a Datos

Implementación del patrón Repository. Abstracción de operaciones de persistencia, consultas mediante SQLAlchemy ORM, aislamiento de la lógica de base de datos.

4. Modelos y Esquemas

  • Modelos SQLAlchemy (entidades de base de datos)
  • Esquemas Pydantic (validación y serialización)
  • Sistema de migraciones con Alembic

5. Servicios Integrados

  • Alpha Vantage API (datos de mercado)
  • OpenAI API (procesamiento de lenguaje natural)
  • PostgreSQL (sistema de persistencia)

Patrones y Técnicas Implementadas

Inyección de dependencias, patrón Repository, autenticación mediante JWT, diseño RESTful, arquitectura en capas con separación de responsabilidades.
